# VPS Affiliate 导购监控平台 — 需求规划

> 目标：打造一个类似 `stock.vpszdm.com` 的 VPS 库存实时监控 + Aff 导购 + TG 推送的完整商业闭环平台。

---

## 一、项目总览

| 维度 | 说明 |
| :--- | :--- |
| **核心价值** | 实时监控热门 VPS 商家库存状态，在补货瞬间推送通知并引导用户通过我们的 Aff 链接购买，赚取佣金 |
| **技术栈** | Node.js (Express) + SQLite + 前端 (HTML/CSS/JS 或 React) + Telegram Bot API |
| **参考竞品** | [stock.vpszdm.com](https://stock.vpszdm.com)、[stock.bwh91.com](https://stock.bwh91.com)、[stock.dmitea.com](https://stock.dmitea.com) |
| **变现方式** | 商家 Affiliate 佣金（搬瓦工约 30%、RackNerd 约 20%、DMIT 等） |

---

## 二、模块拆分与分工

整个项目分为 **4 个核心模块**。根据各模块的技术特征，规划如下分工：

### 分工总览

| 模块 | 核心职责 | 执行方 | 理由 |
| :--- | :--- | :--- | :--- |
| **模块 A：后端监控引擎** | 定时爬虫 + 库存状态判定 + 数据 API | ⭐ **Antigravity 主导** | 涉及与 youhu 现有架构深度整合（Express / cache / scraper 模式），需要精准理解现有代码 |
| **模块 B：前端展示面板** | 库存状态可视化 + 购买引导 UI | ⭐ **Antigravity 主导** | 需要保持与 youhu 一致的设计语言和组件风格，且涉及实时刷新等交互逻辑 |
| **模块 C：Telegram Bot 推送** | 补货通知 + 用户订阅管理 | 🤖 **GPT 处理** | 相对独立的模块，与主站代码耦合度极低。GPT 擅长生成这类"调 API + 状态机"的独立脚本 |
| **模块 D：内容生态（评测/测速）** | SEO 文章、测速页面、产品数据库 | 🤖 **GPT 处理** | 大量文案生成、数据整理、静态页面模板的工作，GPT 的长文本生成能力更适合 |

#### A2. 爬虫与状态判定引擎 (`stockChecker.js`)

| 功能点 | 说明 |
| :--- | :--- |
| **轮询频率** | 默认每 60 秒检测一次（可按商品优先级调整） |
| **探测策略** | HTTP GET 请求商品页面 → 检测 HTML 中是否包含缺货关键字 |
| **反爬对抗** | 随机 User-Agent 轮换 + 请求间隔抖动（±10s）。搬瓦工/DMIT 有 Cloudflare 盾，可能需要 `cloudscraper` 库 |
| **状态翻转检测** | 对比本轮与上一轮缓存状态，仅在 `无货 → 有货` 时触发通知 |
| **容错** | 请求失败 3 次以上标记为"探测异常"状态，不误报 |

#### A3. 数据 API

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/vps/stock` | GET | 返回所有商品的最新库存状态列表 |
| `/api/vps/stock/:productId` | GET | 返回单个商品的详细状态和历史记录 |
| `/api/vps/providers` | GET | 返回所有商家信息 |
| `/api/vps/health` | GET | 监控引擎自身健康状态（运行时间、上次检测时间等） |

> **[架构决策]** 是独立部署一个新的 Node 服务，还是作为 youhu 现有 `server/index.js` 的子模块挂载？
> - **方案 A（推荐）**：独立项目，独立端口。代码隔离，互不影响。部署在同一台 VPS 上即可。
> - **方案 B**：挂载到 youhu 下，共享 Express 实例。好处是少维护一个进程，坏处是耦合。

### 模块 C：Telegram Bot 推送 — 🤖 GPT 处理

#### C1. 核心功能

| 功能 | 说明 |
| :--- | :--- |
| **补货推送** | 检测到 `无货→有货` 翻转时，立即向频道/群组发送结构化消息 |
| **消息格式** | 包含：商品名、配置概要、价格、直达购买链接（带 Aff）、机房信息 |
| **频道模式** | 建立一个公开 Channel，所有订阅者自动收到推送 |
| **用户订阅(进阶)** | 用户可以向 Bot 发送指令订阅特定商家/地区，如 `/subscribe racknerd us_west` |

#### C2. 移交给 GPT 的交付物清单

GPT 需要交付以下独立可运行的代码：

- [ ] `tgBot.js` — Telegram Bot 主逻辑（基于 `node-telegram-bot-api` 或纯 HTTP 调用）
- [ ] `tgNotifier.js` — 纯推送模块，暴露 `notifyStockChange(product)` 函数供模块 A 的引擎调用
- [ ] 消息模板（支持 HTML 格式的 Telegram 富文本）
- [ ] Bot 创建与配置指南文档

> **为什么交给 GPT**：TG Bot 是一个完全独立的 I/O 模块，输入是"商品状态变化事件"，输出是"发一条消息"。它与主站后端仅通过一个函数调用耦合，不需要理解 youhu 的架构。GPT 单轮对话就能生成高质量的完整 Bot 代码。

## 六、当前进度与待办计划 (更新于 4月12日)

### ✅ 我们已经做了什么 (已完成模块)

1. **智能补货监控与全网发现引擎 (Smart Discovery Engine)**
   - 彻底打破了早期“手工填配置”的限制。现在拥有一套完全自主的后台爬虫引擎（`discovery.js`）。
   - **两级深扫机制 (Deep Scan)**：引擎可自动监控竞品库存站（bwh91, dmitea）及各大测评论坛（便宜VPS, VNCoupon, LowEndBox）。先抓取文章列表页，识别出含有 `sale/flash/special` 关键词的促销贴后，自动进入文章详情页提取真正的直达特惠购买链接。
   - **智能上下文提取**：无需人为设置规则，系统会自动分析链接周围的 DOM 结构，智能提炼出产品名称、优惠码以及价格。

2. **全自动监控闭环与管理后台 (Auto-Live & Dashboard)**
   - **秒级自动上架**：对于提取到明确价格信息的特价机型，系统绕过人工确认沙盒，直接入库并秒发 TG 上架播报。
   - **自动重试机制**：对于部分仅探测到 PID 但未拿到价格的“待确认”机型，引擎会在每 4 小时的轮询周期内，利用浏览器重新深入商家后台抓取价格，成功后自动转正。
   - **可视化管理后台**：构建了专属的管理后台界面，可执行一键审核、上下架、编辑价格等操作。

3. **补货实时价格追踪 (Live Price on Restock)**
   - 解决了 2026 年因硬件涨价导致“补货时价格变动”的痛点（`scraper.js`）。
   - 系统检测到商品从“缺货”变为“有货”的瞬间，不依赖数据库的旧缓存价，而是实时从网页上提取最新价格。若价格变动，TG 频道播报中将以高亮删除线对比（如 `⚠️ <s>$16.16/年</s> → 价格已更新`），并将新价格自动写入监控库。

4. **TG 强交互机器人与多重预警 (Telegram Bot UI)**
   - 不再只是单纯的被动发消息机器人。现在 TG 机器人已经变成系统的中枢控制器。
   - 实时响应如 `/stock`（查看当下有货）、`/stats`（查看全站库存概览）、`/discover`（强制全网进行新一轮深扫引擎） 等高级运维质量。

5. **前端 UI 重构与商家智能分组 (Frontend Grouping UI)**
   - `🔥 特价VPS` 专属 Tab 页面已重构，产品不再随机混排，而是基于商家特征（搬瓦工、DMIT、RackNerd）智能聚合分组排列，并带有悬浮式分类标题和独立视觉徽章。
   - 性能极度优化：纯净 Vanilla JS + CSS Grid 渲染，2 分钟自动静默刷新页面局部数据。

### 🧩 使用的核心技术点

- **Puppeteer-Extra + Stealth 插件**：利用无头 Chromium 浏览器模拟真实用户轨迹，强力对抗搬瓦工、DMIT 等商家的 Cloudflare `cf-browser-verification` 盾检测。
- **Cheerio 库**：在全网发现引擎的外围使用，提供极速、轻量级的 jQuery 式 DOM 解析，专破竞品网站和测评帖的文章内容。
- **Cloudscraper**：用于配合获取 HTML，突破部分轻量级 WAF (Web App Firewall) 的反爬封锁。
- **EventBus (事件总线设计模式)**：在 Node.js 中运用 EventEmitter 将 `scraper.js`（纯扫描）与 `tgBot.js`（纯通知）彻底解耦，杜绝了模块间循环加载崩溃的致命问题。
- **CSS-Grid 全局自适应布局**：在前端利用 `grid-column: 1 / -1;` 处理复杂卡片矩阵中的全宽 Header 断点效果，保障大屏小屏极致体验。
- **正则化数据嗅探**：通过 `/App\.cloudcone/i` 或 `/pid=\d+/i` 以及动态货币匹配正则，实现智能提取散落文本中的优惠与金额。
